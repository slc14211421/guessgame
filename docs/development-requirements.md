## 2. 产品概述

“你比我猜”是一款聚会猜词类 Android 应用。用户可在首页选择词语分组，点击开始游戏后进入倒计时，再以横屏形式展示词条。游戏过程中，屏幕可按设置以醒目的跑马灯样式或静态样式展示当前词语，用户可手动切换到下一个词语，并可根据偏好调整背景色、词条文字颜色、出词方式和词条滚动开关。

应用同时支持词条分类和词语管理，用户可自行维护游戏词库。

## 3. 产品目标

1. 支持用户按分类管理猜词词库。
2. 支持用户在指定分类下新增、删除词语。
3. 支持用户选择词语分组并开始猜词游戏。
4. 支持游戏开始前 5 秒倒计时。
5. 支持横屏猜词模式，以跑马灯形式展示词条。
6. 支持用户自定义背景色、词条颜色、出词方式和词条滚动开关，并持久保存设置。
7. 提供简洁、易操作、适合聚会场景的移动端体验。

## 4. 用户角色

| 角色 | 说明 |
| --- | --- |
| 普通用户 | 使用应用选择词语分组、开始游戏、切换词语、调整显示设置。 |
| 词库维护者 | 维护分类和分类下的词语。首版可由普通用户承担该角色。 |

## 5. 术语说明

| 术语 | 说明 |
| --- | --- |
| 词条分类 / 词语分组 | 一组同类型词语，例如成语、水果、蔬菜、动物、人物。 |
| 词语 / 词条 | 游戏中被展示出来供玩家猜测的具体内容。 |
| 猜词模式 | 横屏展示词条并进行游戏的页面状态。 |
| 跑马灯 | 当前词条以横向滚动或循环移动的方式显示。 |
| 用户设置 | 用户选择的背景色、词条颜色、出词方式、词条滚动开关等偏好配置。 |

## 6. 功能范围

### 6.1 首版包含

1. 首页词语分组列表。
2. 词语分类管理。
3. 分类下词语新增、删除。
4. 选择分组后开始游戏。
5. 开始游戏前单选猜词时间：1、2、3、5、10 分钟。
6. 5 秒倒计时和语音读秒提示音。
7. 横屏猜词模式。
8. 猜词模式顶部展示本局剩余时间。
9. 当前词条跑马灯展示。
10. 手动滑动切换下一个词语。
11. 游戏剩余 10 秒提示音和游戏结束音效。
12. 设置入口。
13. 设置背景色、词条颜色、出词方式和词条滚动开关。
14. 设置持久化保存。
15. 退出猜词模式。

### 6.2 首版暂不包含

1. 联网词库同步。
2. 多人账号体系。
3. 排行榜、积分、得分统计。
4. 语音识别判断答题正确。
5. 后台管理系统。
6. 云端备份。
7. 自定义音频素材、复杂音效系统。

## 7. 页面与流程

### 7.1 首页

#### 页面目标

展示所有词语分组，用户选择分组后开始游戏。

#### 页面元素

| 元素 | 说明 |
| --- | --- |
| 原生导航栏标题 | 显示应用名称“你比我猜”，内容区不重复展示同名标题。 |
| 设置图标 | 位于内容区首行右侧，点击后打开设置面板。 |
| 词语分组列表 | 展示所有可选分类。 |
| 分组项 | 显示分类名称、词语数量。 |
| 开始游戏按钮 | 用户选择一个分组后可点击开始。 |
| 分类管理入口 | 进入分类和词语管理页面。 |

#### 交互规则

1. 用户进入首页后默认展示全部词语分组。
2. 用户点击某个分组后，该分组进入选中状态。
3. 未选择分组时，开始游戏按钮不可用或点击后提示“请选择词语分组”。
4. 已选择分组但该分组没有词语时，不允许开始游戏，并提示“该分组暂无词语”。
5. 用户必须单选猜词时间后才能开始游戏，未选择时提示“请选择猜词时间”。
6. 点击开始游戏后进入 5 秒倒计时页面。
7. 点击内容区首行右侧设置图标，打开设置面板。
8. 首页内容区不重复展示与原生导航栏相同的页面标题。

### 7.2 设置面板

#### 页面目标

允许用户配置猜词模式和首页相关的显示偏好。

#### 设置项

| 设置项 | 类型 | 说明 |
| --- | --- | --- |
| 背景色 | 颜色选择 | 设置首页和猜词模式背景色，或至少应用到猜词模式。 |
| 词条颜色 | 颜色选择 | 设置猜词模式中词条文字颜色。 |
| 出词方式 | 单选 | 设置猜词模式按顺序或随机展示词条。 |
| 词条滚动 | 单选 / 开关 | 设置猜词模式中词条是否横向滚动。 |

#### 交互规则

1. 用户修改设置后应立即预览或在确认后生效。
2. 用户设置需要持久化保存，下次打开应用时自动恢复。
3. 若用户从首页打开设置，保存后返回首页。
4. 若用户从猜词模式打开设置，保存后返回猜词模式并立即应用样式。
5. 设置面板需要提供关闭入口。

#### 默认设置

| 设置项 | 默认值 |
| --- | --- |
| 背景色 | 深色或高对比纯色，建议 `#111111` |
| 词条颜色 | 高对比亮色，建议 `#FFFFFF` |
| 出词方式 | 顺序出词 |
| 词条滚动 | 开启 |

### 7.3 分类管理页面

#### 页面目标

维护词条分类。

#### 页面元素

| 元素 | 说明 |
| --- | --- |
| 分类列表 | 展示全部分类。 |
| 新增分类按钮 | 新建一个词条分类。 |
| 删除分类按钮 | 删除指定分类。 |
| 分类详情入口 | 进入该分类下的词语管理。 |

#### 交互规则

1. 分类名称不能为空。
2. 分类名称建议不超过 12 个中文字符或 24 个英文字符。
3. 分类名称不建议重复。若重复，应提示用户修改。
4. 删除分类前需要二次确认。
5. 删除分类时，其下词语一并删除。
6. 至少建议内置 5 个默认分类：成语、水果、蔬菜、动物、人物。

### 7.4 词语管理页面

#### 页面目标

维护某个分类下的词语。

#### 页面元素

| 元素 | 说明 |
| --- | --- |
| 分类名称 | 当前正在维护的分类。 |
| 词语列表 | 展示当前分类下所有词语。 |
| 新增词语入口 | 向当前分类添加词语。 |
| 删除词语入口 | 删除指定词语。 |

#### 交互规则

1. 词语内容不能为空。
2. 单个词语建议不超过 12 个中文字符或 24 个英文字符。
3. 同一分类下词语不建议重复。若重复，应提示用户。
4. 删除词语前可直接删除，也可根据实现成本提供二次确认；若支持批量删除，必须二次确认。
5. 新增、删除成功后列表应立即刷新。

### 7.5 倒计时页面

#### 页面目标

在正式进入猜词模式前给玩家准备时间。

#### 页面元素

| 元素 | 说明 |
| --- | --- |
| 倒计时数字 | 从 5 到 1 显示。 |
| 准备提示 | 可显示“准备开始”。 |
| 读秒音效 | 每秒播放短促提示音。 |

#### 交互规则

1. 用户点击开始游戏后进入倒计时。
2. 倒计时总时长为 5 秒。
3. Android 真机环境下，进入倒计时页后立即切换为横屏。
4. 倒计时页先完整播放“游戏在5秒后开始，请将手机屏幕对着你的队友，加油哦”，播放完成后再开始 5 秒语音读秒。
5. 倒计时结束后自动进入猜词模式。
6. 倒计时期间应支持返回首页，避免误触开始后无法退出；返回首页时恢复竖屏。

### 7.6 猜词模式页面

#### 页面目标

以横屏跑马灯形式展示当前词条，供玩家进行猜词。

#### 页面元素

| 元素 | 说明 |
| --- | --- |
| 当前词条 | 以大字号展示，并支持跑马灯效果。 |
| 剩余时间 | 位于顶部居中，展示本局倒计时。 |
| 退出入口 | 位于右上角，点击后返回首页。 |

#### 横屏规则

1. 进入猜词模式后，应用应切换或锁定为横屏展示。
2. 猜词模式下应用应根据手机横握方向自动切换正反横屏。
3. 退出猜词模式后，应用恢复为正常竖屏使用体验。
4. 若受 uni-app 或系统限制无法强制横屏，应至少对猜词模式进行横屏布局适配。

#### 词条展示规则

1. 当前词条居中或以高识别度区域展示。
2. 词条字号应足够大，适合多人远距离观看。
3. 词条颜色使用用户设置的词条颜色。
4. 背景色使用用户设置的背景色。
5. 当词条较长时，应保证不被截断；可通过跑马灯滚动展示完整内容。
6. 猜词模式期间应保持屏幕常亮，HBuilderX 云打包 APK 不应因系统自动息屏中断游戏。

#### 跑马灯规则

1. 词条滚动开启时，当前词条以横向滚动形式循环展示。
2. 滚动速度应适中，保证可读性。
3. 词条滚动关闭时，当前词条静态居中展示。

#### 游戏计时规则

1. 开始游戏前必须在 1、2、3、5、10 分钟中单选一个猜词时间。
2. 猜词模式顶部居中展示本局剩余时间，字体加粗。
3. 猜词模式不再显示“滑动换词”提示。
4. 游戏剩余 10 秒时开始每秒播放数字倒计时提示音。
5. 游戏时间结束后播放“本局游戏结束，再接再厉哦”，并提示用户返回首页。

#### 切词规则

1. 用户可通过手动滑动进入下一个词语。
2. 推荐支持向左滑动切换到下一个词语。
3. 若当前分类下词语已到最后一个，可循环回到第一个词语。
4. 切换时应避免重复触发，可设置短暂防抖。
5. 词条滚动开启时，切换后跑马灯从新词语重新开始，并在连续切词时左右交替移动方向。

#### 退出规则

1. 用户可从右上角退出入口离开猜词模式。
2. 退出前可直接返回首页，也可弹出确认，首版建议直接返回首页。
3. 退出后不需要记录游戏进度。

## 8. 数据需求

### 8.1 分类数据结构

```json
{
  "id": "category_001",
  "name": "成语",
  "createdAt": "2026-07-10T00:00:00.000Z",
  "updatedAt": "2026-07-10T00:00:00.000Z"
}
```

### 8.2 词语数据结构

```json
{
  "id": "word_001",
  "categoryId": "category_001",
  "text": "画蛇添足",
  "createdAt": "2026-07-10T00:00:00.000Z",
  "updatedAt": "2026-07-10T00:00:00.000Z"
}
```

### 8.3 用户设置数据结构

```json
{
  "backgroundColor": "#111111",
  "wordColor": "#FFFFFF",
  "wordOrder": "sequential",
  "isWordScrollEnabled": true
}
```

### 8.4 本地存储要求

1. 分类、词语、用户设置均需保存在本地。
2. 推荐使用 `uni.setStorageSync` / `uni.getStorageSync` 或封装后的本地存储模块。
3. 首次启动应用时，如果本地没有分类和词语数据，应初始化默认分类。
4. 用户新增、删除分类和词语后，应立即同步到本地存储。
5. 用户设置保存后，下次启动应用必须恢复。

## 9. 默认词库建议

首版建议至少提供以下默认分类。每个分类可先内置少量词语，后续由用户自行维护。

| 分类 | 示例词语 |
| --- | --- |
| 成语 | 画蛇添足、守株待兔、亡羊补牢 |
| 水果 | 苹果、香蕉、西瓜、草莓 |
| 蔬菜 | 黄瓜、白菜、土豆、番茄 |
| 动物 | 老虎、大象、熊猫、兔子 |
| 人物 | 孙悟空、诸葛亮、李白、武则天 |

## 10. 非功能需求

### 10.1 兼容性

1. 支持主流 Android 手机。
2. 建议适配 Android 8.0 及以上版本。
3. 应适配常见手机屏幕比例。
4. 猜词模式需重点适配横屏展示。

### 10.2 性能

1. 首页分组列表加载时间应小于 1 秒。
2. 切换词语应流畅，无明显卡顿。
3. 跑马灯动画应稳定运行。
4. 本地数据读写不应阻塞主要交互。

### 10.3 易用性

1. 首页路径清晰，用户应能在 3 次点击内开始游戏。
2. 猜词模式字体足够大，适合远距离观看。
3. 设置项数量保持克制，首版只保留必要选项。
4. 删除类操作应有明确反馈。

### 10.4 稳定性

1. 本地存储读取失败时，应使用默认数据兜底。
2. 分类或词语为空时，应显示空状态提示。
3. 用户快速滑动切词时不应导致页面异常或词条错乱。

## 11. 权限需求

首版功能不需要申请摄像头、麦克风、定位、通讯录等敏感权限。

为保证云打包 APK 猜词模式不自动息屏，Android 需声明普通权限 `android.permission.WAKE_LOCK`。

若后续增加语音识别、拍照、云同步等功能，再补充对应权限说明。

## 13. 核心业务流程

### 13.1 开始游戏流程

1. 用户打开应用进入首页。
2. 首页加载本地分类、词语和用户设置。
3. 用户选择一个词语分组。
4. 用户单选猜词时间。
5. 用户点击开始游戏。
6. 系统校验该分组下是否存在词语，并校验是否已选择猜词时间。
7. 校验通过后进入倒计时页，Android 真机环境下倒计时页立即切换为横屏。
8. 倒计时页完整播放游戏开始提醒语音后，再开始 5 秒数字读秒。
9. 倒计时结束进入横屏猜词模式。
10. 系统展示该分组下第一个词语，并在顶部展示剩余时间。
11. 用户滑动切换下一个词语。
12. 游戏时间结束后提示用户返回首页。
13. 用户也可从退出入口返回首页。

### 13.2 维护词库流程

1. 用户从首页进入分类管理。
2. 用户新增、删除分类。
3. 用户进入某个分类的词语管理页面。
4. 用户新增、删除词语。
5. 系统保存变更到本地。
6. 用户返回首页后，分组列表和词语数量刷新。

### 13.3 修改设置流程

1. 用户点击首页内容区首行右侧设置图标。
2. 系统打开设置面板。
3. 用户选择背景色和词条颜色。
4. 系统保存设置。
5. 当前页面立即应用设置。
6. 下次启动应用时恢复该设置。

## 14. 异常与空状态

| 场景 | 系统处理 |
| --- | --- |
| 首次启动无本地数据 | 初始化默认分类和默认词语。 |
| 首页无任何分类 | 显示空状态，并引导新增分类。 |
| 分类下无词语 | 禁止开始游戏，并提示用户先添加词语。 |
| 删除分类 | 弹出确认提示，确认后删除分类及其词语。 |
| 删除词语 | 删除后刷新词语列表。 |
| 本地设置不存在 | 使用默认背景色和词条颜色。 |
| 本地数据格式异常 | 使用默认数据兜底，并避免应用崩溃。 |

## 15. 验收标准

### 15.1 首页验收

1. 打开应用后可看到词语分组列表。
2. 首页内容区首行右侧存在设置入口。
3. 用户可选择某个词语分组。
4. 用户可单选 1、2、3、5、10 分钟猜词时间。
5. 点击开始游戏后进入 5 秒倒计时。
6. 未选择分组、分组无词语或未选择猜词时间时，有明确提示。

### 15.2 分类和词语管理验收

1. 用户可新增分类。
2. 用户可删除分类。
3. 用户可进入分类并查看该分类下词语。
4. 用户可新增词语。
5. 用户可删除词语。
6. 重启应用后，用户维护过的分类和词语仍然存在。

### 15.3 倒计时验收

1. 点击开始游戏后显示 5 秒倒计时。
2. Android 真机环境下倒计时页立即横屏展示。
3. 倒计时页完整播放游戏开始语音提醒后，再开始 5 秒数字读秒。
4. 倒计时结束后自动进入猜词模式。
5. 倒计时期间应用无卡死或异常跳转。

### 15.4 猜词模式验收

1. 猜词模式以横屏布局展示。
2. 顶部居中加粗展示本局剩余时间。
3. 当前词条以跑马灯形式展示。
4. 当前词条颜色符合用户设置。
5. 背景色符合用户设置。
6. 用户手动滑动后可进入下一个词语。
7. 最后一个词语后继续滑动可循环到第一个词语。
8. 剩余 10 秒开始播放数字倒计时提示音。
9. 游戏结束后播放结束语音提示并提示返回首页。
10. 用户可退出猜词模式并返回首页。
11. HBuilderX 云打包 APK 中，猜词模式在系统自动息屏时间之后仍保持屏幕常亮。

### 15.5 设置验收

1. 用户可设置背景色。
2. 用户可设置词条颜色。
3. 用户可设置出词方式。
4. 用户可设置词条是否滚动。
5. 设置保存后立即生效。
6. 重启应用后设置仍然生效。

## 16. 开发建议

### 16.1 模块划分

| 模块 | 说明 |
| --- | --- |
| 页面模块 | 首页、分类管理、词语管理、倒计时、猜词模式。 |
| 存储模块 | 统一封装分类、词语、设置的本地读写。 |
| 设置模块 | 管理背景色、词条颜色和默认值。 |
| 词库模块 | 管理分类和词语的增删查。 |
| 游戏模块 | 管理当前分类、当前词条索引、切词逻辑。 |

### 16.2 实现优先级

| 优先级 | 功能 |
| --- | --- |
| P0 | 首页分组列表、开始游戏、倒计时、猜词模式、滑动切词。 |
| P0 | 本地存储、默认词库、设置持久化。 |
| P1 | 分类管理、词语新增和删除。 |
| P1 | 设置面板、背景色和词条颜色选择。 |
| P2 | 更丰富的默认词库、动画细节优化、删除确认体验优化。 |

## 17. 待确认问题

1. 猜词模式中是否需要支持“上一个词语”？
2. 词语展示顺序是固定顺序、随机顺序，还是可由用户选择？
3. 是否需要记录本轮游戏已展示过哪些词语？
4. 是否需要计时器、得分、跳过、答对等游戏功能？
5. 设置中的背景色和词条颜色是否只影响猜词模式，还是同时影响首页？
6. 分类和词语是否需要支持编辑名称，还是首版只做新增和删除？
7. APK 需要适配的最低 Android 版本是否有明确要求？
