## 3. 打包前检查清单

### 3.1 代码和测试

打包前建议在项目根目录执行：

```bash
npm run typecheck
npm test
```

预期结果：

1. TypeScript 类型检查无错误。
2. Vitest 单元测试全部通过。

### 3.2 HBuilderX 项目检查

1. 用 HBuilderX 打开项目根目录 `D:\AI\guessgame`。
2. 确认 HBuilderX 真机运行已经使用当前最新代码。
3. 打开 `src/manifest.json` 的可视化配置界面。
4. 保存一次 manifest，确保可视化配置和 JSON 文件同步。

### 3.3 AppID

当前 AppID 是占位值：

```text
__UNI__GUESSGAME
```

正式打包前建议在 HBuilderX 中执行：

1. 打开 `src/manifest.json`。
2. 进入基础配置。
3. 点击重新获取或绑定 DCloud AppID。
4. 保存 manifest。

说明：

1. 真机测试可临时使用占位 AppID。
2. 正式 APK 建议绑定正式 DCloud AppID，便于后续云打包、版本管理和升级维护。

### 3.4 Android 包名

正式包必须配置稳定的 Android 包名。建议格式：

```text
com.<你的域名或简称>.guessgame
```

示例：

```text
com.example.guessgame
```

注意：

1. `com.example.guessgame` 仅适合测试示例，正式发布时建议替换为自己的唯一包名。
2. 包名一旦用于正式发版，后续升级不要随意修改。
3. 修改包名后，需要重新打包安装验证。

### 3.5 App 图标

在 HBuilderX 中配置：

1. 打开 `src/manifest.json`。
2. 进入 App 图标配置。
3. 选择源图：

```text
generated_images/app-icon.png
```

4. 点击自动生成各尺寸图标。
5. 保存 manifest。

检查点：

1. Android 桌面图标显示清晰。
2. 最近任务列表图标显示正常。
3. 图标无明显裁切、黑边或透明异常。

### 3.6 权限

首版不需要申请以下敏感权限：

1. 摄像头。
2. 麦克风。
3. 定位。
4. 通讯录。
5. 短信。
6. 电话。

HBuilderX 打包界面中不要额外勾选无关模块或权限。

## 4. 证书选择

### 4.1 测试包

如果只是自己安装测试，可使用 HBuilderX 云打包提供的测试证书。

适用场景：

1. 本机测试。
2. 发给少量测试人员体验。
3. 不准备长期分发或上架。

限制：

1. 不建议用于正式发布。
2. 后续改用正式证书时，可能需要先卸载旧测试包再安装正式包。

### 4.2 正式包

如果准备长期分发、交付用户或上架，建议使用自有 Android 签名证书。

需要保存的信息：

1. `.keystore` 文件。
2. 证书别名。
3. 证书密码。
4. 私钥密码。

重要规则：

1. 证书必须长期保存，丢失后会影响后续版本升级。
2. 同一个包名后续升级必须使用同一套签名证书。
3. 不要把证书文件和密码提交到公开仓库。

可选命令示例：

```bash
keytool -genkeypair -v -keystore guessgame-release.keystore -alias guessgame -keyalg RSA -keysize 2048 -validity 36500
```

说明：

1. 如果本机没有 `keytool`，需要安装 JDK。
2. 使用 HBuilderX 云打包时，也可以在打包界面按提示上传已有证书。

## 5. HBuilderX 云打包流程

推荐直接在 HBuilderX 当前项目中操作。

1. 打开项目 `D:\AI\guessgame`。
2. 打开 `src/manifest.json`，确认 AppID、包名、版本号、图标、权限已配置。
3. 顶部菜单选择：

```text
发行 -> 原生App-云打包
```

4. 平台选择 Android。
5. 选择打包类型：
   - 测试安装：可使用测试证书。
   - 正式发布：使用自有证书。
6. 检查模块和权限，不勾选无关能力。
7. 勾选 release 或正式包相关选项。
8. 提交云打包。
9. 打包完成后下载 APK。
10. 安装到 Android 真机进行最终回归。

## 6. CLI 构建产物方式

如果需要先生成 App 构建产物，可在项目根目录执行：

```bash
npm run build:app
```

构建产物位置：

```text
dist/build/app
```

之后可在 HBuilderX 中导入该目录进行后续打包。

说明：

1. 你当前已经在 HBuilderX 中做真机测试，优先使用 HBuilderX 当前项目直接云打包即可。
2. CLI 构建产物方式适合自动化构建或单独交付打包产物时使用。

## 7. 打包后安装验证

APK 下载后，至少完成以下回归。

### 7.1 首次启动

1. 安装 APK。
2. 首次打开应用。
3. 首页显示默认分类。
4. 默认分类显示词语数量。
5. 未选择分类点击开始游戏时提示“请选择词语分组”。

### 7.2 词库管理

1. 新增分类成功。
2. 删除分类前有确认。
3. 删除分类后，该分类下词语同步删除。
4. 进入词语管理页后可新增词语。
5. 删除词语后列表刷新。
6. 杀进程并重新打开后，新增数据仍然存在。

### 7.3 设置

1. 修改背景色后保存。
2. 修改词条颜色后保存。
3. 切换顺序出词或随机出词。
4. 开启或关闭词条滚动。
5. 重启应用后设置仍然保留。

### 7.4 猜词模式

1. 选择有词语的分组。
2. 未选择猜词时间时，开始游戏提示“请选择猜词时间”。
3. 单选 1、2、3、5 或 10 分钟后可以开始游戏。
4. 进入 5 秒倒计时，期间播放读秒提示音。
5. 倒计时结束后播放游戏开始音效，并进入猜词模式。
6. 猜词页横屏展示。
7. 顶部居中加粗展示本局剩余时间。
8. 手机左右横握时，页面方向自动调整且不倒置。
9. 背景色铺满全屏，底部无白边。
10. 词条滚动开启时跑马灯显示正常。
11. 词条滚动关闭时静态居中显示正常。
12. 滑动切词正常。
13. 连续切词时词条左右交替移动。
14. 最后一个词语后继续切词会回到第一个词语。
15. 按音量上键或音量下键可以切词。
16. 音量键切词时不显示系统音量调节条。
17. 游戏剩余 10 秒开始播放倒计时提示音。
18. 游戏结束后播放结束音效，并提示返回首页。
19. 退出猜词模式后恢复竖屏和系统导航栏。

## 9. 常见问题

### 9.1 安装失败

可能原因：

1. 手机上已有相同包名但签名不同的旧包。
2. Android 系统禁止安装未知来源应用。
3. APK 下载不完整。

处理方式：

1. 卸载旧包后重新安装。
2. 开启允许安装未知来源应用。
3. 重新下载 APK。

### 9.2 覆盖安装失败

如果包名相同但签名证书不同，Android 不允许覆盖安装。

处理方式：

1. 测试阶段可卸载旧包后重新安装。
2. 正式发布后必须保持同一包名和同一签名证书。

### 9.3 图标未更新

处理方式：

1. 确认 HBuilderX 已使用 `generated_images/app-icon.png` 自动生成图标。
2. 保存 `src/manifest.json`。
3. 重新云打包。
4. 卸载旧包后安装新包，避免系统缓存旧图标。

### 9.4 音量键切词仍显示音量条

检查点：

1. HBuilderX 版本是否支持 `plus.key.setVolumeButtonEnabled(false)`。
2. 是否安装的是最新打包 APK。
3. 是否在猜词模式页测试，而不是首页或其它页面。
4. 是否使用了自定义基座或旧运行时。

### 9.5 猜词页底部仍有白边

检查点：

1. 是否安装的是最新打包 APK。
2. 猜词模式是否成功进入全屏。
3. 设备是否强制显示系统导航栏。
4. HBuilderX 运行时是否支持 `plus.navigator.hideSystemNavigation()`。
